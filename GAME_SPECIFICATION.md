# ゲーム仕様書 (GAME SPECIFICATION)

本書は、本プロジェクト（DCGプロジェクト）の開発メンバーがゲームの仕組みとロジックを理解するための仕様書です。

---

## 1. 全体概要 (Overview)

本ゲームは、3本のレーン、前後列の概念、および独自のコスト体系（SPと神族レベル）を採用した、戦略的な対戦型カードゲームです。

### 1-1. 勝利条件 (Win Conditions)
-   **ダイレクトアタック成功**: 相手の全てのシールドを破壊した状態で、相手本体にダメージを与える。
-   **デッキアウト（山札切れ）**: 自分のターンのドローフェイズに山札が0枚でドローできない場合、その場で敗北となります。

---

## 2. リソース管理 (Resource Management)

### 2-1. SP (Divine Power / 霊力)
-   カードのプレイや、神族レベルの底上げに使用します。
-   毎ターン開始時に **+3 SP** 獲得します。
-   後攻プレイヤーの第1ターンのみ、ハンデとして **+4 SP** 獲得します。

### 2-2. 神族レベル (Tribe Level)
-   赤・青・緑・白・黒の5つの属性レベルです。
-   **コスト制約**: カードをプレイするには、そのカードの属性レベルが「カードのコスト」以上である必要があります。
-   **レベル上げ**: 1 SPを消費して、任意の属性の神族レベルを 1 上げることができます（最大 Lv.9）。
-   ※レベルは消費されません（カードをプレイしてもレベル数値は維持されます）。

---

## 3. 盤面構成 (Board Layout)

盤面は「3つのレーン」と「前列・後列」で構成されます。

```text
  [ 相手後列 ] [ 相手後列 ] [ 相手後列 ]
  [ 相手前列 ] [ 相手前列 ] [ 相手前列 ]
  ------------------------------------ (境界線)
  [ 自分前列 ] [ 自分前列 ] [ 自分前列 ]
  [ 自分後列 ] [ 自分後列 ] [ 自分後列 ]
```

### 3-1. 配置ルール
-   各スロットには1体のユニットが配置可能です（計 3x2 = 6スロット）。
-   ユニットは配置されたターンは攻撃できません（キーワード「速攻」を除く）。

---

## 4. 戦闘と攻撃ルール (Combat & Attack Rules)

本ゲームの最大の特徴は「正面ユニットによるガード」と「レーン制」です。

### 4-1. 攻撃の優先順位 (Target Priority)
攻撃側ユニットと同じレーンの正面から順にターゲットが決まります。
1.  **正面の敵前列ユニット**: 同レーンに敵がいる場合、まずこれを破壊する必要があります。
2.  **正面の敵後列ユニット**: 同レーンの前列が不在の場合、後列を攻撃可能になります。
3.  **シールド**: 同レーンが完全に空（前列・後列ともに不在）の場合、相手のシールドを攻撃します。
4.  **ダイレクトアタック**: 全てのシールドが破壊され、かつ正面レーンが空の場合、相手本体へダイレクトアタックが可能です。

### 4-2. 前後列の守護関係
-   **前列は後列とシールドを守ります**。
-   「挑発」を持つユニットが前列にいる場合は、他のレーンのターゲットよりも優先して攻撃対象となります。

### 4-3. 戦闘ロジック (Damage Calculation)
-   **相互ダメージ**: 攻撃側がダメージを与えると同時に、防御側も攻撃力分のダメージを「反撃」として与えます。
-   **撃破**: HPが 0 になったユニットは破壊されます。
-   **連撃 (Double Strike)**: 1回目の攻撃で敵を撃破した場合、反撃を受けずに戦闘を終了します。

---

## 5. シールドシステム (Shield System)

-   各プレイヤーはゲーム開始時に **3つのシールド** を持ちます。
-   シールドには「耐久値（Durability）」があり、攻撃を受けると減少します（通常は -1）。
-   シールドの耐久値が 0 になると破壊され、シールドに秘められた **「スキル」** が自動的に発動します。
-   **ライフ計算**: ゲーム画面の「ライフ」は、残りシールドの合計耐久値 + 1（ダイレクトアタック分）として表示されます。

---

## 6. キーワード能力一覧 (Keywords)

| キーワード | 名称 | 効果概要 |
| :--- | :--- | :--- |
| **Taunt** | 挑発 | 前列にいる時、相手はこのユニット以外を攻撃できない。 |
| **Rush** | 速攻 | 場に出たターンから攻撃可能。 |
| **Stealth** | 潜伏 | 攻撃するまで相手の攻撃対象にならない。 |
| **Double Strike** | 連撃 | 攻撃時に2回ダメージを与える。1撃目で倒せば反撃を受けない。 |
| **Barrier** | 加護 | 1度だけダメージを無効化する。 |
| **Endure** | 不屈 | 破壊される時、1度だけHP 1で耐える。 |
| **Siege** | 攻城 | シールドへのダメージが 2 になる。 |
| **Drain** | 吸命 | ダメージを与えた時、自身のHPを最大 2 回復する。 |

---

## 7. カードデータ管理

-   **cards.csv**: ユニット情報のマスターデータ（属性、コスト、ATK/HP、能力）。
-   **shields.csv**: シールド名称と初期耐久値。
-   **shield_skills.csv**: シールド破壊時に発動する効果定義。
